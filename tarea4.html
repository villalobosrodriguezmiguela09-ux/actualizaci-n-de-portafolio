01 · Introducción

¿Qué es CSS3?

El lenguaje que convierte el esqueleto HTML en una experiencia visual.

CSS significa Cascading Style Sheets (Hojas de Estilo en Cascada). Es el lenguaje que le dice al navegador cómo debe verse el contenido HTML: colores, tamaños, fuentes, posiciones y animaciones.

CSS3 es la versión moderna, con características poderosas como gradientes, animaciones, Flexbox, Grid y variables — todo sin necesitar JavaScript.

🎨
¿Para qué sirve?
Controla colores, fuentes, espaciado, diseño y cualquier aspecto visual de una página web.
🌊
¿Qué significa "cascada"?
Las reglas CSS se aplican en cascada: el navegador aplica los estilos de arriba hacia abajo, pudiendo sobreescribir unos a otros.
📁
¿Cómo se vincula con HTML?
Con la etiqueta <link> en el <head> del HTML:
<link rel="stylesheet" href="estilos.css"/>
⚡
3 formas de escribir CSS
Externo: archivo .css separado (recomendado)
Interno: etiqueta <style> en el <head>
En línea: atributo style="" en la etiqueta
💡
Siempre prefiere CSS externo. Crea un archivo estilos.css y vincúlalo con <link>. Así el código queda separado, organizado y reutilizable en varias páginas.

02 · Base

Sintaxis de CSS

La anatomía de una regla CSS.

Una regla CSS tiene tres partes: el selector (a quién aplico el estilo), la propiedad (qué cambio) y el valor (cómo lo cambio).

/* Anatomía de una regla CSS */

h1 {   ← selector
color: #4a90c4;  ← propiedad: valor;
font-size: 2rem;
margin-bottom: 1rem;
}          ← cierre con llave
1
Selector — Apunta al elemento HTML que quieres estilizar. Puede ser una etiqueta (p), una clase (.mi-clase) o un ID (#mi-id).
2
Propiedad — La característica visual que cambias: color, font-size, background, margin, etc. Van en minúsculas y con guión.
3
Valor — El valor que asignas a la propiedad: un color, un número, una unidad, una función. Siempre termina con punto y coma ;.
4
Comentarios — Escribe notas con /* así */. El navegador los ignora pero te ayudan a entender tu código después.
⚠️
Errores comunes: olvidar el punto y coma ; al final de cada propiedad, o las llaves { } que encierran el bloque. Un solo error puede romper todos los estilos siguientes.

Selectores

Elige exactamente qué elementos quieres estilizar.

TipoSintaxisDescripción
Etiquetap { }Aplica a todos los <p> de la página.
Clase.nombre { }Aplica a elementos con class="nombre". El más usado.
ID#nombre { }Aplica a un único elemento con id="nombre".
Universal* { }Aplica a todos los elementos. Útil para reset.
Descendientediv p { }Aplica a los <p> que estén dentro de un <div>.
Hijo directoul > li { }Solo los <li> hijos directos del <ul>.
Pseudo-clasea:hover { }Cuando el mouse pasa sobre un enlace.
Pseudo-claseli:first-child { }Primer elemento de una lista.
Pseudo-claseli:nth-child(2) { }El segundo elemento de una lista.
Pseudo-elementop::before { }Inserta contenido antes del elemento (solo CSS).
Pseudo-elementop::after { }Inserta contenido después del elemento.
Atributoinput[type="text"] { }Aplica solo a los inputs de tipo texto.
Agrupaciónh1, h2, h3 { }Aplica a varios selectores al mismo tiempo.
selectores.css
/* Selector de etiqueta */
p { color: #374151; }

/* Selector de clase (más usado) */
.tarjeta {
  background: white;
  border-radius: 12px;
  padding: 1.5rem;
}

/* Pseudo-clase hover */
.boton:hover {
  background: #4a90c4;
  color: white;
}

/* Pseudo-elemento before */
.titulo::before {
  content: "✦ ";
  color: #e88080;
}

/* Varios selectores juntos */
h1, h2, h3 {
  font-family: 'Space Mono', monospace;
  font-weight: 700;
}

colores.css
/* Todas las formas de definir color */
.caja {
  /* HEX — el más usado */
  color: #4a90c4;
  color: #fff;        /* 3 dígitos = #ffffff */

  /* RGB */
  color: rgb(74, 144, 196);

  /* RGBA — a = opacidad (0 a 1) */
  background: rgba(74, 144, 196, 0.5);

  /* HSL — tono, saturación, luminosidad */
  border-color: hsl(207, 46%, 53%);

  /* Nombre de color en inglés */
  color: steelblue;

  /* Gradiente lineal */
  background: linear-gradient(
    135deg,
    #4a90c4,
    #c44a4a
  );

  /* Gradiente radial */
  background: radial-gradient(
    circle,
    #7fb3e8,
    #1a1a2e
  );
}

05 · Texto

Tipografía

Propiedades para controlar todo el texto de tu página.

tipografia.css
/* Fuente de Google Fonts */
@import url('https://fonts.googleapis.com/css2?family=DM+Sans&display=swap');

body {
  font-family: 'DM Sans', sans-serif;
  font-size: 16px;        /* base */
  line-height: 1.6;       /* interlineado */
  color: #374151;
}

h1 {
  font-size: 2.5rem;     /* rem = relativo al root */
  font-weight: 700;       /* bold */
  letter-spacing: -0.03em; /* negativo = más compacto */
  text-transform: uppercase;
}

p {
  font-size: 1rem;
  text-align: justify;   /* left | center | right | justify */
  text-indent: 2rem;     /* sangría primera línea */
}

.enlace {
  text-decoration: none;  /* quita el subrayado */
  text-decoration: underline;
  text-shadow: 2px 2px 4px rgba(0,0,0,.2);
}

/* Texto que no cabe: cortar con "..." */
.truncar {
  white-space: nowrap;
  overflow: hidden;
  text-overflow: ellipsis;
}

06 · Espaciado

Modelo de caja

Todo elemento HTML es una caja con 4 capas.

El Box Model es el concepto más importante de CSS. Todo elemento HTML es una caja rectangular con 4 capas: contenido, padding, border y margin.

margin
border
padding
content
box-model.css
.caja {
  /* CONTENIDO */
  width: 300px;
  height: 200px;

  /* PADDING — espacio interior */
  padding: 20px;              /* todos los lados */
  padding: 10px 20px;        /* arriba/abajo   izq/der */
  padding-top: 10px;
  padding-right: 20px;
  padding-bottom: 10px;
  padding-left: 20px;

  /* BORDER — borde */
  border: 2px solid #4a90c4;
  border-radius: 12px;       /* esquinas redondeadas */
  border-top: 4px solid #c44a4a;

  /* MARGIN — espacio exterior */
  margin: 20px;
  margin: 0 auto;           /* centra horizontalmente */
  margin-bottom: 1.5rem;

  /* box-sizing: evita que padding/border agrandar la caja */
  box-sizing: border-box;  /* ← SIEMPRE úsalo */

  /* Sombra */
  box-shadow: 0 4px 20px rgba(0,0,0,.1);
}
✅
Reset universal recomendado: Agrega siempre * { box-sizing: border-box; } al inicio de tu CSS. Así el padding y border se incluyen dentro del width/height, sin causar desbordamientos inesperados.

07 · Flujo

Propiedad display

Controla cómo se comporta un elemento en el flujo de la página.

ValorComportamientoEjemplos
blockOcupa todo el ancho, empieza en nueva línea.div, p, h1, section
inlineSolo ocupa el espacio de su contenido, no acepta width/height.span, a, strong, em
inline-blockComo inline pero acepta width, height y margin.img, button
noneEl elemento desaparece completamente del flujo.menús ocultos
flexActiva Flexbox en los hijos directos.navbars, cards
gridActiva CSS Grid en los hijos directos.layouts de página
display.css
/* Ocultar un elemento */
.menu-movil  { display: none; }

/* Centrar un bloque */
.contenedor {
  display: block;
  width: 800px;
  margin: 0 auto;  /* centra */
}

/* Inline-block para botones en línea */
.etiqueta {
  display: inline-block;
  padding: 4px 12px;
  border-radius: 999px;
  background: #deedf9;
}

08 · Diseño

Flexbox

El sistema de diseño unidimensional más poderoso de CSS.

Flexbox organiza elementos en una fila o columna. Perfecta para navbars, listas de tarjetas, centrar elementos y distribuir espacio.

flexbox.css
.contenedor {
  display: flex;

  /* Dirección */
  flex-direction: row;          /* row | column | row-reverse */

  /* Alineación horizontal (eje principal) */
  justify-content: space-between; /* flex-start | center | space-around */

  /* Alineación vertical (eje cruzado) */
  align-items: center;          /* flex-start | flex-end | stretch */

  /* Espacio entre elementos */
  gap: 1rem;

  /* Permitir que los elementos se envuelvan */
  flex-wrap: wrap;
}

/* Elementos hijos */
.item {
  flex: 1;         /* crecer y ocupar espacio disponible */
  flex: 0 0 200px; /* ancho fijo de 200px */
  align-self: flex-end; /* alineación individual */
}

/* Truco: centrar perfectamente */
.centrado {
  display: flex;
  justify-content: center;
  align-items: center;
}

09 · Diseño 2D

CSS Grid

El sistema de cuadrículas bidimensional para layouts complejos.

CSS Grid organiza elementos en filas y columnas al mismo tiempo. Ideal para layouts de página completos, galerías de imágenes y dashboards.

grid.css
.grid {
  display: grid;

  /* 3 columnas iguales */
  grid-template-columns: 1fr 1fr 1fr;

  /* Shorthand: repeat(3, 1fr) */
  grid-template-columns: repeat(3, 1fr);

  /* Auto-fill: se adapta al espacio disponible */
  grid-template-columns: repeat(auto-fill, minmax(250px, 1fr));

  /* Filas */
  grid-template-rows: 100px auto 60px;

  /* Espacio entre celdas */
  gap: 1rem;
  column-gap: 1rem;
  row-gap: 2rem;
}

/* Elemento que ocupa 2 columnas */
.elemento-grande {
  grid-column: span 2;
  grid-row: span 2;    /* ocupa 2 filas */
}

/* Layout con áreas nombradas */
.layout {
  display: grid;
  grid-template-areas:
    "header header"
    "sidebar main"
    "footer footer";
  grid-template-columns: 200px 1fr;
}
.header  { grid-area: header; }
.sidebar { grid-area: sidebar; }
.main    { grid-area: main; }
.footer  { grid-area: footer; }

10 · Posición

Position

Controla la posición exacta de cualquier elemento.

ValorDescripción
staticDefault. El elemento sigue el flujo normal del documento. No responde a top/left/right/bottom.
relativeSe mueve relativo a su posición original. Sirve de referencia para elementos absolute dentro.
absoluteSale del flujo. Se posiciona relativo al padre con position: relative más cercano.
fixedSe posiciona relativo a la ventana del navegador. No se mueve al hacer scroll. Para navbars fijas.
stickyCombina relative y fixed: se comporta como relative hasta llegar a un punto, luego se queda fijo.
posicion.css
/* Navbar fija en la parte superior */
.navbar {
  position: fixed;
  top: 0;
  left: 0;
  right: 0;
  z-index: 100;       /* encima de todo */
}

/* Badge encima de una imagen */
.tarjeta   { position: relative; }
.badge {
  position: absolute;
  top: 10px;
  right: 10px;
  background: #c44a4a;
  color: white;
  border-radius: 999px;
  padding: 4px 10px;
}

/* Encabezado que se queda pegado */
.encabezado-seccion {
  position: sticky;
  top: 60px;          /* se pega a 60px del top */
}

11 · Movimiento suave

Transiciones

Agrega suavidad cuando las propiedades cambian de valor.

Las transiciones animan el cambio entre dos estados de un elemento (por ejemplo, al hacer hover). Sin transición el cambio es instantáneo; con transición es suave.

transiciones.css
.boton {
  background: #4a90c4;
  color: white;
  padding: 10px 24px;
  border-radius: 8px;
  border: none;
  cursor: pointer;

  /* transition: propiedad duración timing */
  transition: all 0.3s ease;

  /* O especifica sólo lo que cambia: */
  transition:
    background 0.3s ease,
    transform 0.2s ease,
    box-shadow 0.3s ease;
}

.boton:hover {
  background: #c44a4a;
  transform: translateY(-4px);
  box-shadow: 0 8px 20px rgba(196,74,74,.3);
}

/* Funciones de temporización */
/* ease | linear | ease-in | ease-out | ease-in-out */
/* cubic-bezier(.4, 0, .2, 1)  — personalizado */
Pasa el mouse por los botones

12 · Movimiento

Animaciones

Crea movimientos automáticos con @keyframes.

Mientras las transiciones reaccionan a interacciones, las animaciones se ejecutan solas, en bucle o en secuencia. Se definen con @keyframes y se aplican con animation.

animaciones.css
/* 1. Define la animación con @keyframes */
@keyframes flotar {
  0%   { transform: translateY(0); }
  50%  { transform: translateY(-20px); }
  100% { transform: translateY(0); }
}

@keyframes aparecer {
  from { opacity: 0; transform: translateY(30px); }
  to   { opacity: 1; transform: translateY(0); }
}

@keyframes girar {
  to { transform: rotate(360deg); }
}

/* 2. Aplica la animación */
.flotante {
  /* nombre  duración  timing  delay  repetición  dirección  modo-relleno */
  animation: flotar 3s ease-in-out 0s infinite normal both;
}

.entrada {
  animation: aparecer 0.6s ease both;
}

/* Múltiples animaciones */
.complejo {
  animation:
    girar 2s linear infinite,
    flotar 4s ease-in-out infinite;
}

/* Pausa al hover */
.complejo:hover {
  animation-play-state: paused;
}

13 · Organización

Variables CSS

Custom Properties: reutiliza valores y crea temas en segundos.

Las variables CSS (Custom Properties) te permiten guardar un valor una vez y usarlo en todo el archivo. Si necesitas cambiar un color o fuente, lo cambias en un solo lugar.

variables.css
/* Definir variables en :root (disponibles en toda la página) */
:root {
  /* Colores */
  --color-primario: #4a90c4;
  --color-secundario: #c44a4a;
  --color-fondo: #f8f6f2;
  --color-texto: #1a1a2e;

  /* Tipografía */
  --fuente-principal: 'DM Sans', sans-serif;
  --fuente-mono: 'Space Mono', monospace;
  --tamaño-base: 1rem;

  /* Espaciado */
  --radio: 12px;
  --sombra: 0 4px 20px rgba(0,0,0,.1);
}

/* Usar variables con var() */
.tarjeta {
  background: var(--color-fondo);
  border-radius: var(--radio);
  box-shadow: var(--sombra);
  font-family: var(--fuente-principal);
  color: var(--color-texto);
}

.boton-primario {
  background: var(--color-primario);
}
.boton-secundario {
  background: var(--color-secundario);
}

/* Modo oscuro con una sola línea */
[data-tema="oscuro"] {
  --color-fondo: #1a1a2e;
  --color-texto: #f8f6f2;
}
✅
¿Por qué son tan útiles? Si tu cliente pide cambiar el color principal de azul a verde, solo cambias una línea en :root y toda la página se actualiza automáticamente.

14 · Mobile First

Diseño Responsivo

Que tu página se vea bien en celular, tablet y escritorio.

El diseño responsivo hace que una sola página se adapte a cualquier tamaño de pantalla. La técnica principal son las media queries.

responsivo.css
/* ── Mobile First: empieza con móvil ── */
.grid-tarjetas {
  display: grid;
  grid-template-columns: 1fr;    /* 1 columna en móvil */
  gap: 1rem;
}

/* Tablet (≥ 768px) */
@media (min-width: 768px) {
  .grid-tarjetas {
    grid-template-columns: repeat(2, 1fr); /* 2 columnas */
  }
}

/* Desktop (≥ 1024px) */
@media (min-width: 1024px) {
  .grid-tarjetas {
    grid-template-columns: repeat(3, 1fr); /* 3 columnas */
  }
}

/* Pantalla grande (≥ 1280px) */
@media (min-width: 1280px) {
  .contenedor {
    max-width: 1200px;
    margin: 0 auto;
  }
}

/* Ocultar en móvil, mostrar en desktop */
.solo-desktop { display: none; }
@media (min-width: 1024px) {
  .solo-desktop { display: block; }
}

/* Imágenes responsivas */
img {
  max-width: 100%;
  height: auto;
}

/* clamp: tamaño fluido sin media queries */
h1 {
  font-size: clamp(1.8rem, 5vw, 4rem);
  /* mín    escalable    máx */
}
📱
Mobile First
Diseña primero para móvil y ve agregando estilos para pantallas más grandes con min-width.

Breakpoints comunes

15 · Profesional

Buenas prácticas

Hábitos que marcan la diferencia entre código amateur y profesional.

✅Siempre empieza con un reset: * { box-sizing: border-box; margin: 0; padding: 0; }
✅Define tus colores y fuentes como variables en :root al inicio del archivo.
✅Usa clases para estilizar, no IDs. Los IDs quedan para JS y anclas.
✅Organiza tu CSS en bloques: reset → variables → tipografía → layout → componentes → utilidades.
✅Escribe CSS Mobile First: primero el diseño móvil, luego agrega media queries para pantallas más grandes.
✅Usa rem para tamaños de fuente y espaciado, no px fijos. Así respetas las preferencias del usuario.
✅Usa transition: all .3s ease para darle vida a los elementos interactivos.
✅Agrega comentarios para separar secciones: /* ── NAVBAR ── */
✅Prueba en varios navegadores: Chrome, Firefox y Safari pueden comportarse diferente.
✅Usa las DevTools (F12) constantemente para inspeccionar y probar estilos en tiempo real.
❌No uses !important salvo emergencias. Rompe la cascada y dificulta el mantenimiento.
❌No pongas todos los estilos en línea (style=""). Imposible mantener y poco semántico.
❌No uses tablas para layout. Para eso están Flexbox y Grid.
