01 · Introducción

¿Qué es HTML5?

El punto de partida de toda página web en el mundo.

HTML significa HyperText Markup Language (Lenguaje de Marcado de Hipertexto). No es un lenguaje de programación — es un lenguaje de estructura: le dice al navegador cómo organizar y mostrar el contenido de una página web.

HTML5 es la versión más reciente y actual del estándar, publicada en 2014 por el W3C. Trajo consigo etiquetas semánticas, soporte nativo para audio y video, formularios mejorados y mucho más.

🌐
¿Para qué sirve?
Define la estructura y el contenido de una página: títulos, párrafos, imágenes, enlaces, tablas y formularios.
🧱
¿Cómo funciona?
Usamos etiquetas (tags) que el navegador interpreta y convierte en la página visual que el usuario ve.
🤝
HTML + CSS + JS
HTML pone el esqueleto, CSS le da el estilo y JavaScript añade la interactividad.
⚡
¿Qué necesito?
Solo un editor de texto (VS Code es el mejor) y un navegador web. ¡Nada más!
💡
Consejo: Descarga Visual Studio Code (gratis en code.visualstudio.com). Instala la extensión Live Server para ver tus cambios en tiempo real sin recargar manualmente.

02 · Fundamentos

Estructura básica

El esqueleto que toda página HTML debe tener, siempre.

Cada archivo HTML debe tener esta estructura mínima. Piensa en ella como el hueso de toda página web. Cópiala y pégala cada vez que empieces un nuevo archivo.

<!-- Declara que es HTML5 -->
<!DOCTYPE html>

<html lang="es"> <!-- Raíz del documento -->
<head> <!-- Información invisible -->
<meta charset="UTF-8"/>
<meta name="viewport" content="width=device-width"/>
<title>Mi primera página</title>
</head>
<body> <!-- Contenido visible -->
<h1>¡Hola, mundo!</h1>
<p>Mi primera página web.</p>
</body>
</html>

Analicemos cada parte:

1
<!DOCTYPE html> No es una etiqueta HTML como las demás. Le avisa al navegador que el documento usa HTML5. Siempre va en la primera línea, sin excepción.
2
<html lang="es"> Es la etiqueta raíz que contiene todo. El atributo lang="es" indica que el idioma es español, lo que ayuda a lectores de pantalla y al SEO.
3
<head> Contiene información sobre la página que el usuario NO ve directamente: el título de la pestaña, estilos CSS, fuentes, etc.
4
<meta charset="UTF-8"/> Permite mostrar caracteres especiales: acentos (á, é), ñ, emojis, etc. Siempre inclúyela.
5
<body> Todo lo que el usuario ve en la página va aquí: textos, imágenes, botones, videos, tablas…
💡
Truco rápido en VS Code: Crea un archivo .html, escribe ! y presiona Tab. VS Code genera automáticamente toda la estructura básica en un segundo.
index.html 
<!DOCTYPE html>
<html lang="es">
<head>
  <meta charset="UTF-8"/>
  <meta name="viewport" content="width=device-width, initial-scale=1.0"/>
  <title>Mi primera página</title>
</head>
<body>
  <h1>¡Hola, mundo!</h1>
  <p>Esta es mi primera página web.</p>
</body>
</html>

03 · Contenido

Etiquetas de texto

Los elementos más usados para mostrar contenido escrito.

Las etiquetas de texto son las más utilizadas en HTML. Con ellas organizamos títulos, párrafos y damos énfasis a palabras clave.

EtiquetaPara qué sirveEjemplo
<h1>…<h6>Títulos de mayor (<h1>) a menor (<h6>) importancia. Solo usa un h1 por página.<h1>Título</h1>
<p>Párrafos de texto. Agrega espacio arriba y abajo automáticamente.<p>Texto</p>
<strong>Texto en negrita con importancia semántica.<strong>importante</strong>
<em>Texto en cursiva con énfasis semántico.<em>énfasis</em>
<br>Salto de línea. No tiene etiqueta de cierre.<br>
<hr>Línea horizontal divisora. No tiene etiqueta de cierre.<hr>
<span>Contenedor en línea para aplicar estilos a una parte del texto.<span>parte</span>
<mark>Resalta texto como si usaras un marcador amarillo.<mark>destacado</mark>
<small>Texto de tamaño pequeño, para notas al pie o aclaraciones.<small>nota</small>
<sub> / <sup>Subíndice (H₂O) y superíndice (x²) respectivamente.<sup>2</sup>
<blockquote>Cita larga en bloque, con sangría.<blockquote>cita</blockquote>
<code>Texto con fuente monoespaciada para código inline.<code>let x=1</code>
textos.html 
<h1>Título principal</h1>
<h2>Subtítulo</h2>
<h3>Sección</h3>

<p>Este es un párrafo normal.</p>

<p>
  Esto es <strong>muy importante</strong>
   y esto tiene <em>énfasis</em>.
</p>

<p>Texto <mark>resaltado</mark> aquí.</p>

<p>Agua = H<sub>2</sub>O</p>
<p>x<sup>2</sup> + y<sup>2</sup></p>

<hr>

<blockquote>
  "El conocimiento es poder" — Francis Bacon
</blockquote>

Ojo: Nunca uses <h1> solo para hacer el texto grande. Las etiquetas h1–h6 tienen un significado semántico: representan jerarquía de información. Para tamaños usa CSS.

04 · Navegación

Listas y enlaces

Organiza información y conecta páginas entre sí.

Las listas agrupan elementos relacionados. Los enlaces <a> son el núcleo del concepto "hiper" en hipertexto: conectan páginas, secciones, archivos y recursos externos.

listas.html 
<!-- Lista sin orden (bullets) -->
<ul>
  <li>HTML</li>
  <li>CSS</li>
  <li>JavaScript</li>
</ul>

<!-- Lista ordenada (números) -->
<ol>
  <li>Aprende HTML</li>
  <li>Aprende CSS</li>
  <li>Aprende JavaScript</li>
</ol>

<!-- Lista de definición -->
<dl>
  <dt>HTML</dt>
  <dd>Lenguaje de marcado</dd>
</dl>

<!-- ENLACES -->
<a href="https://google.com">Ir a Google</a>

<!-- Abrir en nueva pestaña -->
<a href="https://google.com" target="_blank">
  Google (nueva pestaña)
</a>

<!-- Enlace a otra página local -->
<a href="contacto.html">Contacto</a>

<!-- Enlace a sección de la misma página -->
<a href="#cap5">Ir al capítulo 5</a>

<!-- Enlace de email -->
<a href="mailto:hola@correo.com">Escríbeme</a>
Resultado en el navegador
Lista sin orden:
  • HTML
  • CSS
  • JavaScript
Lista ordenada:
  1. Aprende HTML
  2. Aprende CSS
  3. Aprende JavaScript
Enlace: Ir a Google
💡
Atributos importantes del enlace <a>:

href URL de destino
target="_blank" Abre en nueva pestaña
rel="noopener" Seguridad al usar _blank
title="..." Tooltip al pasar el mouse

05 · Multimedia

Imágenes y multimedia

Fotos, audio y video directamente en el navegador.

multimedia.html 
<!-- Imagen básica -->
<img
  src="foto.jpg"
  alt="Descripción de la imagen"
  width="400"
  height="300"
/>

<!-- Imagen responsiva (recomendado) -->
<img
  src="foto.jpg"
  alt="Mi foto"
  style="max-width:100%; height:auto;"
/>

<!-- Imagen con pie de foto -->
<figure>
  <img src="foto.jpg" alt="Paisaje"/>
  <figcaption>Un hermoso paisaje.</figcaption>
</figure>

<!-- VIDEO HTML5 -->
<video controls width="640">
  <source src="video.mp4" type="video/mp4"/>
  Tu navegador no soporta video.
</video>

<!-- AUDIO HTML5 -->
<audio controls>
  <source src="audio.mp3" type="audio/mpeg"/>
</audio>

<!-- VIDEO de YouTube -->
<iframe
  width="560" height="315"
  src="https://www.youtube.com/embed/ID"
  allowfullscreen
></iframe>
⚠️
El atributo alt es obligatorio en <img>. Describe la imagen para personas con discapacidad visual y para cuando la imagen no carga. Si la imagen es decorativa, usa alt="".
💡
Atributos de <video> y <audio> en HTML5:

controls Muestra los controles de reproducción
autoplay Reproduce automáticamente
loop Repite el video/audio
muted Silenciado por defecto
poster="img.jpg" Imagen de previsualización del video

06 · Datos

Tablas

Organiza datos en filas y columnas.

Las tablas se usan para mostrar datos tabulares, como horarios, comparaciones o calificaciones. Nunca las uses para crear layouts de página (eso lo hace CSS).

tabla.html 
<table border="1">

  <!-- Cabecera -->
  <thead>
    <tr>
      <th>Materia</th>
      <th>Calificación</th>
      <th>Estado</th>
    </tr>
  </thead>

  <!-- Cuerpo -->
  <tbody>
    <tr>
      <td>Cultura Digital</td>
      <td>9.5</td>
      <td>✅ Aprobado</td>
    </tr>
    <tr>
      <td>Matemáticas</td>
      <td>8.0</td>
      <td>✅ Aprobado</td>
    </tr>
  </tbody>

  <!-- Pie de tabla -->
  <tfoot>
    <tr>
      <td colspan="2">Promedio general</td>
      <td>8.75</td>
    </tr>
  </tfoot>

</table>

07 · Interacción

Formularios

Recibe información del usuario: texto, opciones, archivos.

Los formularios son la forma principal en que los usuarios interactúan con una web: buscar, iniciar sesión, registrarse, enviar mensajes. HTML5 introdujo nuevos tipos de input muy útiles.

formulario.html 
<form action="procesar.php" method="POST">

  <!-- Texto -->
  <label for="nombre">Nombre:</label>
  <input type="text" id="nombre" name="nombre"
         placeholder="Tu nombre" required/>

  <!-- Email -->
  <input type="email" placeholder="correo@email.com"/>

  <!-- Contraseña -->
  <input type="password" minlength="8"/>

  <!-- Número -->
  <input type="number" min="1" max="100"/>

  <!-- Fecha (nuevo en HTML5) -->
  <input type="date"/>

  <!-- Color picker (nuevo en HTML5) -->
  <input type="color"/>

  <!-- Rango / slider -->
  <input type="range" min="0" max="100"/>

  <!-- Área de texto -->
  <textarea rows="4" cols="40"></textarea>

  <!-- Selección -->
  <select name="grado">
    <option value="1">1° Semestre</option>
    <option value="2" selected>2° Semestre</option>
  </select>

  <!-- Checkbox -->
  <input type="checkbox" id="acepto"/>
  <label for="acepto">Acepto términos</label>

  <!-- Radio buttons -->
  <input type="radio" name="sexo" value="M"/> Masculino
  <input type="radio" name="sexo" value="F"/> Femenino

  <!-- Subir archivo -->
  <input type="file" accept=".jpg,.png"/>

  <!-- Botón de envío -->
  <button type="submit">Enviar</button>
  <button type="reset">Limpiar</button>

</form>
✅
HTML5 valida automáticamente: Al usar type="email" el navegador verifica que el formato sea correcto antes de enviar. Con required no permite enviar si el campo está vacío. ¡Sin JavaScript!

08 · Arquitectura

Etiquetas semánticas

HTML5 llegó con etiquetas que describen el significado del contenido.

Antes de HTML5 todo se hacía con <div>. Las etiquetas semánticas le dicen al navegador, buscadores y lectores de pantalla qué tipo de contenido contiene cada sección.

<body>
<header> ← Encabezado del sitio (logo, menú)
<nav> ← Menú de navegación
<main> ← Contenido principal (solo uno por página)
<article> ← Contenido independiente (un blog post)
<section> ← Sección temática
<aside> ← Barra lateral / info relacionada
</article>
</main>
<footer> ← Pie de página (créditos, links)
</body>
EtiquetaUso recomendado
<header>Encabezado del sitio o de una sección. Logo, título y menú principal.
<nav>Bloque de enlaces de navegación principal.
<main>Contenido principal único de la página. Solo uno por documento.
<section>Agrupación temática de contenido, normalmente con un título.
<article>Contenido independiente y reutilizable: post, noticia, tarjeta.
<aside>Información relacionada pero secundaria. Barras laterales, tips.
<footer>Pie de página con créditos, redes sociales, copyright.
<figure>Contenedor para imagen/gráfico con su descripción <figcaption>.
<time>Representa una fecha/hora legible para máquinas.
<details>Contenido expandible/colapsable sin JavaScript.
✅
¿Por qué es importante la semántica? Google usa las etiquetas semánticas para entender tu página y posicionarla mejor en buscadores (SEO). Los lectores de pantalla las usan para ayudar a personas con discapacidad visual a navegar la web.

09 · Atributos

Atributos globales

Atributos que puedes usar en cualquier etiqueta HTML.

Los atributos son información extra que agregamos dentro de la etiqueta de apertura. Los atributos globales funcionan en cualquier etiqueta HTML.

atributos.html 
<!-- id: identificador único en la página -->
<section id="contacto">…</section>

<!-- class: grupo de elementos para CSS/JS -->
<p class="destacado azul">Texto importante</p>

<!-- style: CSS en línea (evitar si es posible) -->
<p style="color:red; font-size:20px;">Rojo</p>

<!-- title: tooltip al pasar el mouse -->
<abbr title="HyperText Markup Language">HTML</abbr>

<!-- hidden: oculta el elemento -->
<p hidden>No visible</p>

<!-- tabindex: orden del foco con teclado -->
<button tabindex="1">Primero</button>

<!-- contenteditable: texto editable -->
<div contenteditable="true">
  ¡Puedes editar este texto!
</div>

<!-- data-*: atributos personalizados -->
<button data-id="42" data-accion="borrar">
  Eliminar
</button>
🎯
id vs class
id es único (solo un elemento por página). class puede repetirse en varios elementos. Usa class para estilos e id para anclas y JS.
🗂️
data-*
Guarda datos personalizados en el HTML para usar con JavaScript. Ej: data-id="5" luego accesible con element.dataset.id.

10 · Profesional

Buenas prácticas

Los hábitos que separan al principiante del desarrollador profesional.

✅Siempre escribe el <!DOCTYPE html> en la primera línea.
✅Usa minúsculas en todas tus etiquetas y atributos: <p> no <P>.
✅Cierra siempre tus etiquetas. Si abriste <div>, ciérralo con </div>.
✅Usa sangría (indentación) de 2 espacios para que el código sea legible.
✅Agrega el atributo alt a todas tus imágenes.
✅Usa etiquetas semánticas en lugar de llenar todo de <div>.
✅Escribe comentarios (<!-- ... -->) para explicar partes complejas.
✅Valida tu HTML en validator.w3.org antes de publicarlo.
✅Separa la estructura (HTML) del estilo (CSS) y la lógica (JavaScript).
✅Incluye siempre la meta viewport para que tu página sea responsiva en móvil.
❌No uses estilos en línea (style="") en producción. Usa archivos CSS separados.
❌No uses <b> o <i> solo para estilo visual. Usa <strong> y <em> con significado semántico.
❌No uses tablas para diseño de layout. Las tablas son solo para datos tabulares.
