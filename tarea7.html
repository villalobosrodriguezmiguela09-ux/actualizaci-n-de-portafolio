01 · INTRODUCCIÓN

¿QUÉ ES ARDUINO?

La plataforma de electrónica de código abierto que democratizó el hardware.

Arduino es una plataforma de hardware y software de código abierto que combina una placa de microcontrolador con un entorno de programación sencillo. Fue creada en 2005 en Italia y hoy es usada por millones de personas para crear proyectos de electrónica, robótica, automatización e IoT.

🤖
Robótica
Controla motores, servos y brazos robóticos. Es el cerebro de muchos robots educativos y competencias.
🏠
Domótica
Automatiza tu hogar: luces inteligentes, alarmas, sistemas de riego y control de temperatura.
🌱
IoT
Conecta sensores al internet para monitorear temperatura, humedad y enviar datos a la nube en tiempo real.
🎨
Arte interactivo
Crea instalaciones artísticas, instrumentos musicales y esculturas que reaccionan al movimiento o sonido.
✅
Código abierto: Todo el hardware de Arduino es de diseño libre. Puedes descargar los esquemas, fabricar tu propia placa o comprar clones económicos. El software también es gratuito y funciona en Windows, Mac y Linux.

03 · COMPONENTES

COMPONENTES BÁSICOS

Los elementos electrónicos esenciales que usarás en casi todo proyecto.

🔴
COMP 01
LED
SALIDA DIGITAL
Diodo emisor de luz. Tiene ánodo (+) y cátodo (−). Siempre usar con una resistencia de 220Ω–470Ω para no quemarlo.
1.8–3.3V · 20mA máximo
〰️
COMP 02
Resistencia
PASIVO
Limita el flujo de corriente. Se identifica por sus bandas de colores. Las más usadas: 220Ω (LED), 10kΩ (pull-up), 1kΩ (general).
Ohm = Voltaje / Corriente
🔘
COMP 03
Push Button
ENTRADA DIGITAL
Botón pulsador momentáneo. Al presionarlo cierra el circuito. Usar con resistencia pull-up o pull-down de 10kΩ para lectura estable.
4 terminales · SPST momentáneo
🌀
COMP 04
Potenciómetro
ENTRADA ANALÓGICA
Resistencia variable con perilla. Genera un voltaje entre 0V y 5V. Perfecto para controlar brillo, velocidad o volumen con lectura analógica.
10kΩ · 3 terminales
📢
COMP 05
Buzzer
SALIDA DIGITAL
Genera sonido al aplicar voltaje. Los piezoeléctricos son los más sencillos. Con PWM puedes generar tonos y melodías simples.
5V · 30mA · Activo o Pasivo
🔋
COMP 06
Condensador
PASIVO
Almacena y libera energía eléctrica. Se usa para filtrar ruido en la alimentación y estabilizar señales. Respeta la polaridad en electrolíticos.
100µF desacoplamiento · 0.1µF filtro
⚠️
Nunca conectes un LED sin resistencia. Los pines de Arduino dan 5V y el LED solo aguanta ~2V. Sin resistencia, en segundos el LED se quema y puede dañar el pin. La fórmula: R = (5V - 2V) / 0.02A = 150Ω mínimo, usa 220Ω.

El entorno de desarrollo integrado gratuito para programar tu placa.

06 · PROGRAMACIÓN

ESTRUCTURA DEL CÓDIGO

Todo sketch de Arduino tiene exactamente dos funciones obligatorias.

El código de Arduino se llama "sketch" y está basado en C++ simplificado. La estructura es siempre la misma: dos funciones que el microcontrolador ejecuta automáticamente.

estructura_basica.ino
// Zona de declaraciones globales
#define LED_PIN 13       // constante con nombre
int contador = 0;       // variable global

// ── setup(): se ejecuta UNA SOLA VEZ al inicio ──
void setup() {
  pinMode(LED_PIN, OUTPUT);   // configura el pin
  Serial.begin(9600);         // inicia comunicación serial
  Serial.println("¡Listo!");   // envía mensaje
}

// ── loop(): se repite PARA SIEMPRE ──
void loop() {
  digitalWrite(LED_PIN, HIGH); // enciende LED
  delay(1000);                  // espera 1000ms (1 segundo)
  digitalWrite(LED_PIN, LOW);  // apaga LED
  delay(1000);                  // espera 1 segundo

  contador++;                    // incrementa
  Serial.print("Parpadeo #");
  Serial.println(contador);
}

boton_led.ino
// Botón controla LED — lógica básica
const int BOTON = 2;
const int LED   = 13;

void setup() {
  pinMode(BOTON, INPUT);    // entrada
  pinMode(LED,   OUTPUT);   // salida
  Serial.begin(9600);
}

void loop() {
  int estado = digitalRead(BOTON);  // HIGH o LOW

  if (estado == HIGH) {
    digitalWrite(LED, HIGH);
    Serial.println("Botón presionado");
  } else {
    digitalWrite(LED, LOW);
  }
}

// Con INPUT_PULLUP (más sencillo — lógica invertida)
void setup() {
  pinMode(BOTON, INPUT_PULLUP); // resistencia interna
  // Con INPUT_PULLUP: LOW = presionado, HIGH = suelto
}
💡
INPUT_PULLUP es la forma más sencilla de leer botones. Activa una resistencia interna de 20kΩ del Arduino — ya no necesitas la resistencia externa. El botón va entre el pin y GND. Lógica invertida: LOW = presionado.

09 · ADC

SEÑALES ANALÓGICAS

Lee valores variables como temperatura, luz o posición.

analogico.ino
// Potenciómetro en A0 — lectura analógica
const int POT = A0;

void setup() {
  Serial.begin(9600);
}

void loop() {
  int valor = analogRead(POT);     // 0–1023

  // Convertir a voltaje (0–5V)
  float voltaje = valor * 5.0 / 1023.0;

  // Mapear a rango personalizado (ej: 0–180 para servo)
  int angulo = map(valor, 0, 1023, 0, 180);

  Serial.print("ADC: "); Serial.print(valor);
  Serial.print(" | V: "); Serial.print(voltaje);
  Serial.print(" | °: "); Serial.println(angulo);

  delay(200);
}
Función map() — clave en Arduino
map(valor, fromLow, fromHigh, toLow, toHigh)
// Ejemplos prácticos:
map(val, 0, 1023, 0, 255) // ADC → PWM (brillo LED)
map(val, 0, 1023, 0, 180) // ADC → ángulo servo
map(temp, 0, 100, 0, 255) // temperatura → brillo

10 · PWM

SEÑALES PWM

Modulación por ancho de pulso — la forma de simular salidas analógicas.

Arduino UNO es un microcontrolador digital — sus pines solo conocen 0V o 5V. PWM simula voltajes intermedios encendiendo y apagando muy rápidamente el pin. La función analogWrite() acepta valores de 0 (0%) a 255 (100%).

pwm_ejemplos.ino
// Fade de LED (brillo gradual)
const int LED = 9;   // ← DEBE ser pin PWM (~)

void setup() {
  pinMode(LED, OUTPUT);
}

void loop() {
  // Aumentar brillo
  for (int i = 0; i <= 255; i++) {
    analogWrite(LED, i);
    delay(8);
  }
  // Disminuir brillo
  for (int i = 255; i >= 0; i--) {
    analogWrite(LED, i);
    delay(8);
  }
}

// Control de brillo con potenciómetro
void loop() {
  int pot    = analogRead(A0);         // 0–1023
  int brillo = map(pot, 0, 1023, 0, 255); // 0–255
  analogWrite(LED, brillo);
}

sensor_ultrasonido.ino
// HC-SR04 — Sensor de distancia ultrasónico
const int TRIG = 9;   // Pin Trigger
const int ECHO = 10;  // Pin Echo

void setup() {
  pinMode(TRIG, OUTPUT);
  pinMode(ECHO, INPUT);
  Serial.begin(9600);
}

void loop() {
  // 1. Enviar pulso ultrasónico
  digitalWrite(TRIG, LOW);
  delayMicroseconds(2);
  digitalWrite(TRIG, HIGH);
  delayMicroseconds(10);
  digitalWrite(TRIG, LOW);

  // 2. Medir tiempo de regreso
  long duracion = pulseIn(ECHO, HIGH);

  // 3. Calcular distancia (velocidad sonido = 343 m/s)
  float distancia = duracion * 0.034 / 2;

  Serial.print("Distancia: ");
  Serial.print(distancia);
  Serial.println(" cm");
  delay(100);
}

MONITOR SERIAL

Comunica Arduino con tu computadora para depurar y enviar datos.

serial_comunicacion.ino
// Serial Monitor — tu ventana al interior de Arduino
void setup() {
  Serial.begin(9600);   // velocidad en baudios
  Serial.println("=== Arduino listo ===");
}

void loop() {
  int sensor = analogRead(A0);

  // Diferentes formas de imprimir
  Serial.print("Sensor: ");    // sin salto de línea
  Serial.println(sensor);      // con salto de línea

  // Recibir datos desde el monitor serial
  if (Serial.available() > 0) {
    String dato = Serial.readString();
    dato.trim();    // elimina espacios y \n

    if (dato == "ON")
      digitalWrite(13, HIGH);
    else if (dato == "OFF")
      digitalWrite(13, LOW);
  }

  delay(500);
}

15 · PROFESIONAL

BUENAS PRÁCTICAS

Hábitos que separan al hobbyist del maker profesional.

✅
Usa #define o const int para nombrar tus pines. Nunca uses números "mágicos" en el código: const int LED = 13; es mucho más claro que solo 13.
✅
Siempre usa resistencias con LEDs. Calcula con la ley de Ohm: R = (Vcc - Vled) / Iled. Para LED rojo a 5V: R = (5-2) / 0.02 = 150Ω (usa 220Ω por seguridad).
✅
Prueba el circuito en breadboard antes de soldarlo permanentemente. La breadboard permite hacer y deshacer conexiones sin daño.
✅
Usa colores de cable estándar: rojo = positivo (5V/VCC), negro = negativo (GND), otros colores = señal. Facilita depurar circuitos complejos.
✅
Comenta tu código con //. Describe qué hace cada bloque, qué valor esperas de cada sensor y qué significan los números "mágicos.
✅
Usa millis() en lugar de delay() en proyectos complejos. delay() bloquea todo el programa; millis() permite hacer varias cosas a la vez.
✅
Instala librerías desde el Gestor de Librerías del IDE (Croquis → Incluir Librería). Nunca copies archivos .h manualmente a menos que sea necesario.
✅
Revisa el Monitor Serial para depurar. Imprime el valor de sensores y variables con Serial.println() para entender qué está pasando.
❌
Nunca conectes directamente un motor DC al pin de Arduino. Los motores consumen demasiada corriente y pueden quemar el microcontrolador. Usa siempre un driver como el L298N.
❌
No uses delay() si necesitas leer sensores o botones durante la espera. Bloquea todo el programa. Usa la técnica de millis() para temporización sin bloqueo.
❌
No excedas la corriente máxima de los pines (40mA por pin, 200mA total). Para más corriente usa transistores, MOSFETs o módulos de potencia.
