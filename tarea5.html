// 02 · Estructura HTML

Todo documento HTML tiene la misma estructura base. Piensa en ella como el esqueleto de tu página — siempre debe estar completa, sin importar qué tan sencilla o compleja sea la página.

<!-- 1. Declaración HTML5 -->
<!DOCTYPE html>

<html lang="es"> <!-- 2. Raíz -->
<head> <!-- 3. Cabeza (invisible) -->
<meta charset="UTF-8"/>
<meta name="viewport" content="width=device-width"/>
<title>Mi Página</title>
<link rel="stylesheet" href="estilos.css"/>
</head>
<body> <!-- 4. Cuerpo (visible) -->
<header>…encabezado…</header>
<main>…contenido…</main>
<footer>…pie…</footer>
</body>
</html>
1
<!DOCTYPE html> Le dice al navegador que el documento usa HTML5. Siempre en la primera línea, sin excepción.
2
<html lang="es"> La etiqueta raíz que envuelve todo. El atributo lang="es" indica que el idioma es español.
3
<head> Contiene información sobre la página que el usuario NO ve: título de la pestaña, fuentes, estilos CSS.
4
<body> Todo lo que el usuario SÍ ve va aquí: textos, imágenes, botones, formularios, tablas.
index.html — estructura completa 
<!DOCTYPE html>
<html lang="es">
<head>
  <meta charset="UTF-8"/>
  <meta name="viewport" content="width=device-width, initial-scale=1.0"/>

  </main>

  <footer>
    <p>© 2025 Mi Nombre</p>
  </footer>

</body>
</html>

// 03 · Estructura CSS

Una regla CSS tiene tres partes: el selector (a quién aplico el estilo), la propiedad (qué cambio) y el valor (cómo lo cambio). Todo va encerrado entre llaves { }.

/* Anatomía de una regla CSS */

h1 { ← selector (apunta al <h1>)
color : #4a90c4 ; ← propiedad: valor;
font-size : 2rem ;
margin-bottom : 1rem ;
} ← cierre de la regla
estilos.css — estructura completa 
/* ── 1. RESET BÁSICO ── */
* {
  margin: 0;
  padding: 0;
  box-sizing: border-box;
}

/* ── 2. VARIABLES ── */
:root {
  --color-primario: #4a90c4;
  --color-secundario: #c44a4a;
  --fuente: 'DM Sans', sans-serif;
  --radio: 12px;
}

/* ── 3. TIPOGRAFÍA BASE ── */
body {
  font-family: var(--fuente);
  font-size: 16px;
  line-height: 1.6;
  color: #374151;
  background: #f8f6f2;
}

/* ── 4. TÍTULOS ── */
h1 { font-size: 2.5rem; color: var(--color-primario); }
h2 { font-size: 1.8rem; margin-bottom: 1rem; }
h3 { font-size: 1.2rem; }

/* ── 5. CLASES REUTILIZABLES ── */
.contenedor {
  max-width: 1100px;
  margin: 0 auto;
  padding: 0 1.5rem;
}

.tarjeta {
  background: white;
  border-radius: var(--radio);
  padding: 1.5rem;
  box-shadow: 0 4px 20px rgba(0,0,0,.08);
}

/* ── 6. INTERACTIVIDAD ── */
.boton {
  background: var(--color-primario);
  color: white;
  padding: 10px 24px;
  border: none;
  border-radius: 8px;
  cursor: pointer;
  transition: all .3s ease;
}
.boton:hover {
  background: var(--color-secundario);
  transform: translateY(-3px);
}

box-model.css 
.mi-caja {
  /* Tamaño del contenido */
  width: 300px;
  height: 150px;

  /* Padding — espacio interior */
  padding: 20px;             /* todos lados */
  padding: 10px 20px;       /* vertical horizontal */

  /* Border — borde visible */
  border: 2px solid #4a90c4;
  border-radius: 12px;      /* esquinas redondeadas */

  /* Margin — espacio exterior */
  margin: 20px;
  margin: 0 auto;           /* centrar horizontalmente */

  /* Sombra */
  box-shadow: 0 4px 20px rgba(0,0,0,.1);
}
// 05 · Layout con Flexbox

Flexbox es el sistema más sencillo para organizar elementos en fila o columna. Con una sola propiedad (display: flex) transformas cómo se colocan los elementos hijos.

flexbox.css 
.fila {
  display: flex;
  justify-content: space-between; /* distribución horizontal */
  align-items: center;            /* alineación vertical */
  gap: 1rem;                       /* espacio entre hijos */
  flex-wrap: wrap;                 /* se adapta al ancho */
}

/* Centrar perfectamente */
.centrado {
  display: flex;
  justify-content: center;
  align-items: center;
}

/* Hijo que ocupa todo el espacio disponible */
.hijo-grande { flex: 1; }

// 07 · Consejos para aprender más rápido
🖊️
Escribe el código a mano. No copies y pegues al principio. Escribir cada etiqueta y propiedad ayuda a tu cerebro a memorizarlas más rápido.
🔁
Practica todos los días aunque sea 20 minutos. La consistencia es más importante que practicar 3 horas un solo día.
🔍
Usa F12 (DevTools). Inspecciona cualquier página web para ver su código HTML y CSS. Es la mejor forma de aprender de páginas reales.
🧪
Experimenta y rompe cosas. Cambia valores, borra etiquetas y observa qué pasa. Aprenderás más del error que del éxito.
📋
Ten a la mano una hoja de referencia. No memorices todo desde el principio. Consultar es normal y hasta los profesionales lo hacen.
🌱
Empieza simple. Haz primero el HTML sin estilos. Cuando el contenido esté correcto, agrega el CSS. No intentes hacer todo al mismo tiempo.
💬
Usa comentarios en tu código. Escribe <!-- esto es un nav --> en HTML o /* botón principal */ en CSS para recordar para qué sirve cada parte.
📱
Prueba en celular. Usa DevTools (F12 → ícono de celular) para ver cómo se ve tu página en diferentes tamaños de pantalla.
🎮
Juega Flexbox Froggy y Grid Garden. Son juegos gratis en el navegador que te enseñan Flexbox y CSS Grid de forma visual e interactiva.
📖
Consulta MDN Web Docs. Es la documentación oficial de HTML y CSS en español. Busca cualquier etiqueta o propiedad en developer.mozilla.org/es
⚠️
Error #1 más común: Olvidar cerrar etiquetas HTML. Si abres <div> debes cerrarlo con </div>. Usa la indentación para visualizar mejor la jerarquía.
⚠️
Error #2 más común en CSS: Olvidar el punto y coma ; al final de cada propiedad. Un solo punto y coma faltante puede hacer que toda la regla no funcione.
✅
Validador oficial: Pega tu código HTML en validator.w3.org y tu CSS en jigsaw.w3.org/css-validator para que el W3C te diga si hay errores.
🐰
Consejo secreto: Los mejores desarrolladores no saben todo de memoria — saben cómo buscar. Google + MDN + ChatGPT son tus mejores herramientas. ¡Úsalas sin pena!
// Reflexión
✦ ¿Qué aprendí con esta práctica?

Con esta práctica entendí que HTML y CSS trabajan en equipo: HTML pone la estructura y CSS le da vida visual. Lo más importante fue familiarizarme con la forma de escribir el código: las etiquetas, las llaves, los dos puntos y los puntos y coma tienen un lugar exacto. También aprendí que el Box Model es fundamental para entender cómo se posiciona todo en una página, y que Flexbox hace mucho más sencillo el trabajo de organizar elementos. El siguiente paso es practicar creando páginas reales combinando todo lo aprendido.
